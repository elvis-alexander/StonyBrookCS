Your hipster friends are thinking about starting a new coffee shop (Star Duck Coffee). They are looking for a computer based method to keep track of orders. They have asked you to make a system that will keep track of orders, where they can add orders to the end of the list, to the front of the list (if they get a big tip), or in the middle of a list (if they get a medium sized tip). They would like to be able to delete orders (in case somebody gets bored and leaves, or they run out of organic soymilk), edit orders, move them around (copy them, move the cursor, and paste), and switch orders between them. Also, for efficiency, they would like to be able to put orders of the same type next to each other (because nobody wants to peel a guava for Guava Spice Latte twice). You will be implementing an order manager using linked lists. You will account for two baristas, each of which has a linked list of orders, and a cursor.

/*

Required Classes

Order

Write a fully-documented class named Order which contains the name of the order (String), special instruction to create the order (String), and the price of the order in dollars (double). You should provide getter and setter methods for all member variables. In addition, you should provide a constructor as detailed below, though you may create a custom constructor which takes any arguments you see fit. Lastly, you should provide a toString() method that returns a printable representation of the Order and its data members (name, specialInstruction, price).

  • private String order
  • private String specialInstruction
  • private double price
  • public Order(String name, double price)
  • Default constructor.
  • Postconditions:
  • This object has  been initialized to an order object with specified name and price).
  • public boolean equals(Object obj)
  • Compare this Order to another object for equality.
  • Returns a value of true if the obj refers to an Order object with the same properties as this Order. Otherwise, returns a value of false.

OrderListNode

Write a fully-documented class named OrderListNode that wraps an Order object to allow it to be inserted into a doubly linked-list data structure. The Order object reference should be contained in a field called data and there should be two OrderListNode references serving as ‘links’ to the previous and next OrderListNodes in the list. You should provide getter and setter methods for all member variables in this class as well. The class will be based on the following ADT specification:

  • private Order data
  • private OrderListNode next
  • private OrderListNode prev
  • public OrderListNode(Order initData)
  • Default constructor.
  • Preconditions:
  • initData is not null.
  • Postconditions:
  • This OrderListNode has been initialized to wrap the indicated Order, and prev and next have been set to null.
  • Throws:
  • IllegalArgumentException if initData is null.

OrderList

Write a fully-documented class named OrderList which implements a double linked-list data structure. The OrderList should maintain a list of Orders by chaining a series of OrderListNodes between a head and a tail reference. In addition, a cursor should be provided to allow a user to traverse the list, selecting individual OrderListNodes to allow for insertion, deletion, and manipulation of the Orders they contain. Lastly, the class should provide methods to query information about the list, such as its size and the total price of all the Orders in the list.

  • private OrderListNode head
  • private OrderListNode tail
  • private OrderListNode cursor
  • public OrderList()
  • Default constructor which initializes this object to an empty list of Orders.
  • Postconditions:
  • This OrderList has been initialized with head, tail, and cursor all set to null.
  • public int numOrders()
  • Returns the total number of Orders in the list.
  • This method should run in O(1) time.
  • public Order getCursorOrder()
  • Gets the reference to the Order wrapped by the OrderListNode currently referenced by cursor.
  • Returns the reference by the Order wrapped by the OrderListNode currently referenced by cursor. If the cursor is null, then this method should return null as well (i.e. the cursor does not reference an Order).
  • public void resetCursorToHead()
  • Returns the cursor to the starat of the list.
  • Postconditions:
  • If head is not null, the cursor now references the first OrderListNode in this list.
  • If head is null, the cursor is set to null as well (there are no Orders in this list).
  • public void cursorForward()
  • Moves the cursor to select the next OrderListNode in the list. If the cursor is at the tail of the list, this method throws an exception (this includes the case where cursor and tail are both null).
  • Throws:
  • EndOfListException if cursor is at the tail of the list.
  • public void cursorBackward()
  • Moves the cursor to select the previous OrderListNode in the list. If the cursor is at the head of the list, this method throws an exception (this includes the caase where cursor and head are both null).
  • Throws:
  • EndOfListException if cursor is at the head of the list.
  • public void inserAfterCursor(Order newOrder)
  • Inserts the indicated Order after the cursor.
  • Preconditions:
  • newOrder is not null.
  • Postconditions:
  • newOrder has been wrapped in a new OrderListNode object.
  • If cursor was previously not null, the newly created OrderListNode has been inserted into the list before the cursor.
  • If cursor was previously null, the newly created OrderListNode has been set as the new head of the list (as well as the tail).
  • The cursor now references the newly created OrderListNode.
  • Throws:
  • IllegalArgumentException if newOrder is null.
  • Warning:
  • You should NOT move data references around between OrderListNodes to accomplish this method. Instead, you should wrap the newOrder object in a new OrderListNode object, and insert this object into the list after the cursor.
  • public void appendToTail(Order newOrder)
  • Inserts the indicated Order after the tail of the list.
  • Preconditions:
  • newOrder is not null.
  • Postconditions:
  • newOrder has been wrapped in a new OrderListNode object.
  • If tail was previously not null, the newly created OrderListNode has been inserted into the list after the tail.
  • If tail was previously null, the newly created OrderListNode has been set as the new head of the list (as well as the tail and the cursor).
  • The tail now references the newly created OrderListNode.
  • Throws:
  • IllegalArgumentException if newOrder is null.
  • Note:
  • This insertion method does not affect the cursor, unless the list was previously empty. In that case, head, tail, and cursor should all reference the new OrderListNode.
  • public Order removeCursor()
  • Removes the OrderListNode referenced by cursor and returns the Order inside.
  • Preconditions:
  • cursor is not null.
  • Postconditions:
  • The OrderListNode referencedby cursor has been removed from the list.
  • All other OrderListNodes in the list exist in the same order as before.
  • The cursor now references the previous OrderListNode (or the head, if the cursor previously referenced the head of the list).
  • Throws:
  • EndOfListException if cursor is null.

Write a fully-documented class named CoffeeOrderManager which creates an instance of the OrderList class and provides an interface for a user to manipulate the list by adding, removing, and editing slides. The following functionality should be provided:

  • public static void main(String[] args)
  • The main method runs a menu driven application which first creates an empty OrderList and then prompts the user for a menu command selecting the operation. The required information is then requested from the user based on the selected operation. You can find the list of menu options in the UI required functions section.

EndOfListException

An Exception class which indicates that the user attempted to access a OrderListNode that does not exist (either before the head node or after the tail node). This exception can also be thrown when an operation is performed on an empty list (i.e. head, tail, and cursor are all equal to null).
